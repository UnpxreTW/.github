# 貢獻指南 · Contributing

## 繁體中文

1. **Fork** 該儲存庫，建立 topic 分支，push 後對 default 分支開 PR。
2. **分支命名**：kebab-case（例如 `add-login-form`、`fix-cache-leak`）。
3. **Commit 訊息**：開頭加 `modify` / `fix` / `feat` / `clean` / `docs` / `refactor` 其一，冒號後接一行祈使句摘要。
4. 各儲存庫的具體規範請見其 README。

### Commit prefix 對照

| Prefix | 用途 |
|--------|------|
| `modify` | 修改、增強既有功能 |
| `fix` | 修 bug |
| `feat` | 新功能 |
| `clean` | 清理 / 移除過時內容 |
| `docs` | 僅文件變更 |
| `refactor` | 重構（無行為變更） |

## English

1. **Fork** the repository, create a topic branch, push, and open a PR against the default branch.
2. **Branch name**: kebab-case (e.g. `add-login-form`, `fix-cache-leak`).
3. **Commit message**: prefix with one of `modify` / `fix` / `feat` / `clean` / `docs` / `refactor`, followed by `:` and a one-line summary in the imperative mood.
4. Refer to each repository's README for project-specific conventions.
